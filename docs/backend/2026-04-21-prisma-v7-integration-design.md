# Prisma v7 Integration Design — Chessium Backend

**Date:** 2026-04-21  
**Scope:** Phase 1 MVP — NestJS backend, PostgreSQL, email/password auth only (no guests)  
**Package manager:** npm

---

## 1. Architecture & Module Layout

### Principle
`PrismaModule` là module hạ tầng dùng chung, export `PrismaService`. Các domain module (`AuthModule`, `GameModule`, `HistoryModule`) chỉ inject `PrismaService` — không gọi Prisma CLI hay config trực tiếp.

### File layout

```
backend/
├── prisma.config.ts            # Prisma v7: defineConfig() — root level, cùng package.json
├── prisma/
│   ├── schema.prisma
│   ├── migrations/             # Auto-generated by prisma migrate dev
│   └── seed.ts                 # Dev seed: sample users + 1 completed game
├── src/
│   ├── prisma/
│   │   ├── prisma.module.ts
│   │   ├── prisma.service.ts
│   │   └── prisma-exception.filter.ts
│   ├── auth/                   # AuthModule: register, login, JWT
│   ├── game/                   # GameModule: chess logic, Socket.io gateway
│   └── history/                # HistoryModule: match history, leaderboard
```

### `prisma.config.ts` (Prisma v7 style)

```typescript
import { defineConfig, env } from 'prisma/config'

export default defineConfig({
  schema: 'prisma/schema.prisma',
  migrations: {
    path: 'prisma/migrations',
  },
  datasource: {
    url: env('DATABASE_URL'),
  },
})
```

```typescript

  async onModuleDestroy() { await this.$disconnect() }

```

## 2. Data Model

### Enums

```prisma
enum GameStatus {
  WAITING
  IN_PROGRESS
  COMPLETED
  ABANDONED
}

enum GameResult {
  WHITE_WIN
  BLACK_WIN
  DRAW
  ONGOING
}

enum EndReason {
  CHECKMATE
  STALEMATE
  TIMEOUT
  RESIGNATION
  DRAW_AGREEMENT
  INSUFFICIENT_MATERIAL
}

enum PieceColor {
  WHITE
  BLACK
}
```

### `User`

```prisma
model User {
  id           String   @id @default(uuid())
  username     String   @unique
  email        String   @unique      // non-nullable: email/password auth only
  passwordHash String                // non-nullable: bcrypt hash
  eloRating    Int      @default(1200)

  gamesAsWhite Game[]   @relation("WhitePlayer")
  gamesAsBlack Game[]   @relation("BlackPlayer")

  createdAt     DateTime @default(now())
  updatedAt     DateTime @updatedAt

  @@index([eloRating])
}
```

> **Removed từ schema gốc:** `isGuest`, nullable `email`, nullable `passwordHash`.  
> **Optional mở rộng Phase 2+:** thêm `emailVerifiedAt DateTime?` khi cần email verification.

### `Game`

```prisma
model Game {
  id          String     @id @default(uuid())
  slug        String     @unique   // Room ID chia sẻ với đối thủ

  initialTime Int        @default(600)  // giây (default 10 phút rapid)
  increment   Int        @default(0)    // giây cộng mỗi nước

  status      GameStatus @default(WAITING)
  result      GameResult @default(ONGOING)
  endReason   EndReason?

  currentFen  String     @default("rnbqkbnr/pppppppp/8/8/8/8/PPPPPPPP/RNBQKBNR w KQkq - 0 1")
  pgn         String?    @db.Text  // nullable: null khi WAITING/IN_PROGRESS, điền khi COMPLETED

  whitePlayerId         String?
  whitePlayer           User?    @relation("WhitePlayer", fields: [whitePlayerId], references: [id])

  blackPlayerId         String?
  blackPlayer           User?    @relation("BlackPlayer", fields: [blackPlayerId], references: [id])

  // Elo snapshot tại thời điểm kết thúc ván (phục vụ history/leaderboard)
  ratingSnapshotWhite   Int?
  ratingSnapshotBlack   Int?

  moves     Move[]

  startTime DateTime?
  endTime   DateTime?
  createdAt DateTime  @default(now())
  updatedAt DateTime  @updatedAt   // thêm để audit state changes

  @@index([status])
  @@index([slug])
}
```

### `Move`

```prisma
model Move {
  id          String     @id @default(uuid())
  gameId      String
  game        Game       @relation(fields: [gameId], references: [id])

  moveNumber  Int
  playerColor PieceColor  // enum thay vì String để tránh typo
  notation    String      // SAN, ví dụ: "e4", "Nf3"
  fenBefore   String
  fenAfter    String

  timeLeft    Int         // ms còn lại của người chơi sau nước đi
  createdAt   DateTime    @default(now())

  @@index([gameId, moveNumber])
}
```

## 3. Data Flow

### Register / Login
```
AuthModule → bcrypt hash password → PrismaService.user.create()
           → issue JWT { userId, username }
```

### Create Game
```
GameModule → PrismaService.game.create({ status: WAITING, whitePlayerId, slug })
           → trả slug về client để share
```

### Join Game
```
GameModule → PrismaService.game.update({ blackPlayerId, status: IN_PROGRESS, startTime })
           → broadcast game_started qua Socket.io
```

### Move (server-authoritative)
```
Client emit move_attempt
→ server validate bằng chess.js adapter
→ nếu hợp lệ:
    prisma.$transaction([
      move.create({ gameId, moveNumber, playerColor, notation, fenBefore, fenAfter, timeLeft }),
      game.update({ currentFen, pgn }),
    ])
→ broadcast move_confirmed { fen, pgn, timeLeft }
→ nếu không hợp lệ: emit move_rejected
```

### Game End
```
Server detect checkmate / timeout / resign / draw
→ prisma.$transaction([
    game.update({ status: COMPLETED, result, endReason, endTime,
                  ratingSnapshotWhite, ratingSnapshotBlack }),
    user.update(white, { eloRating: newWhiteElo }),
    user.update(black, { eloRating: newBlackElo }),
  ])
→ broadcast game_over { result, endReason, newRatings }
```

## 4. Error Handling

### `PrismaExceptionFilter` (global, đặt tại `src/prisma/`)

| Prisma Error Code | HTTP Exception | Trường hợp |
|---|---|---|
| `P2002` (unique violation) | `ConflictException 409` | Email/username/slug trùng |
| `P2025` (record not found) | `NotFoundException 404` | Game/User không tồn tại |
| Khác | `InternalServerErrorException 500` | Log và ẩn chi tiết |

- Filter đăng ký global trong `main.ts` qua `app.useGlobalFilters()`.
- Không xử lý Prisma error rải rác ở từng service.
- Transaction thất bại → rollback toàn bộ, server emit `move_rejected` / `action_failed`.

## 5. Migration Strategy

| Môi trường | Lệnh | Ghi chú |
|---|---|---|
| Dev | `npm run prisma:migrate:dev -- --name <tên>` | Sinh file migration, áp ngay |
| CI / Test | `npm run prisma:migrate:deploy` | Chỉ chạy migrations đã commit |
| Production | `npm run prisma:migrate:deploy` | Deterministic, không auto-generate |

- Đặt tên migration rõ ràng: `init_schema`, `add_rating_snapshot`, `add_email_verified_at`.
- File `.env` không commit; dùng `.env.example` với placeholder `DATABASE_URL=`.
- `TEST_DATABASE_URL` riêng cho e2e test, chạy `migrate deploy` trước suite.

## 6. Testing Strategy

### Unit test
- Mock `PrismaService` bằng `jest.fn()` / jest manual mock.
- Không cần DB thật; test business logic thuần túy ở service layer.

### Integration / E2E test
- DB PostgreSQL riêng (`TEST_DATABASE_URL`).
- Chạy `prisma migrate deploy` trước toàn bộ suite.
- Dọn dữ liệu sau mỗi test bằng `prisma.$transaction([deleteMany...])` theo thứ tự dependency (Move → Game → User).
